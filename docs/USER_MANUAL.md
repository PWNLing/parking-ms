# 用户手册 (USER_MANUAL.md)

## 1. 系统简介

停车场管理信息系统（ParkingMS）是基于 Qt6 + SQLite + HyperLPR3 的桌面单机应用，实现停车场出入库管理、车牌识别、车位可视化、历史查询、账户管理与用户预约功能。

## 2. 首次启动

1. 运行 `./parking_ms`
2. 弹出初始化向导，填写：
   - **停车场信息**：名称（如"新工停车场"）、单价（如 5.00 元/小时）、容量（如 100）
   - **初始管理员账户**：账号（如 admin）、密码（≥6位）、确认密码、真实姓名、手机号（11位）
   - **HyperLPR3 模型路径**：默认已填入用户本机路径，如不同请点击"浏览"选择
3. 点击「完成初始化」
4. 系统自动创建数据库 `data/parking.db` 与配置 `config.json`
5. 进入角色选择

## 3. 角色选择

启动后选择角色：
- **管理员模式**：需登录，管理出入库、查询、账户
- **用户模式**：无需登录，查询车位、预约

两个模式可随时切换（不退出应用）。

## 4. 管理员模式

### 4.1 登录

输入账号密码登录。密码框按回车等同点击登录。无账号可点击「注册」创建。

### 4.2 主页

主页布局：
```
┌─ 标题栏：停车场名 | 单价 | 当前时间 ───────┐
├──────────────────┬─────────────────────────┤
│  摄像头画面       │     车位饼图            │
│  (实时识别叠加框) │  入库/预约/剩余         │
│  [选择本地图片识别]│                        │
├──────────────────┴─────────────────────────┤
│  识别结果：津B6H920 | 蓝牌 | 0.97  [入库][出库]│
├────────────────────────────────────────────┤
│ [车库信息] [账户管理] [修改密码] [退出登录] │
└────────────────────────────────────────────┘
```

**操作流程**：
1. 车辆驶入摄像头视野 → 自动识别 → 识别结果栏显示车牌/类型/置信度
2. 系统自动判断动作（在场中→高亮"出库"；不在场→高亮"入库"）
3. 管理员点击对应按钮确认

**无摄像头时**：点击「选择本地图片识别」选择 jpg/png 图片识别。

**车牌不合法时**：弹出手动输入对话框，管理员输入正确车牌。

### 4.3 车库信息查询

点击「车库信息」进入查询页：
- **全部记录**：默认加载，按入库时间倒序
- **按车牌模糊查询**：输入"津B"匹配所有津B开头的车牌
- **按时间段查询**：选择起止时间
- **组合查询**：车牌 + 时间段同时填写
- **删除记录**：选中行后点击「删除选中记录」（在场中记录会联动 P_now_count -1）

> 表格所有列不可编辑（计费记录涉及费用一致性）。

### 4.4 账户管理

点击「账户管理」进入：
- **添加用户**：在末尾空行输入信息，点「修改选中」会弹出密码输入框，确认后添加
- **修改用户**：双击单元格编辑姓名/手机号，点「修改选中」保存
- **删除用户**：选中行点「删除选中」（不能删除当前登录账户）
- **重置密码**：选中行点「重置密码」，输入新密码

密码列始终显示 MD5 密文（只读）。

### 4.5 修改密码

点击「修改密码」：
- 输入旧密码 + 新密码 + 确认新密码
- 成功后需重新登录

### 4.6 退出登录

点击「退出登录」返回角色选择（不退出应用）。

## 5. 用户模式

### 5.1 查询车位

进入用户模式自动显示停车场状态：
- 总车位 / 已入库 / 已预约
- **剩余车位**（大字突出显示，满时变红）

### 5.2 预约车位

1. 在「我的车牌」输入框填入车牌号（首次输入，下次自动填充）
2. 点击「预约车位」
3. 成功后状态栏提示「预约成功！请于 30 分钟内入场」

**预约规则**：
- 同车牌不可重复预约
- 车位满时不可预约
- 30 分钟未入场自动失效（系统每 60 秒清理一次）
- 入场后预约自动消除

### 5.3 切换角色

点击「切换至管理员模式」返回角色选择。

## 6. 计费规则

- 停车时长 ≤ 30 分钟：**免费**
- 停车时长 > 30 分钟：`单价 × ceil(分钟数 / 60)`（不足 1 小时按 1 小时算）
- 单价取入库时的快照（后续改价不影响历史账单）

示例：
- 入库 12:30，出库 12:50（20分钟）→ 免费
- 入库 12:30，出库 14:10（100分钟）→ ceil(100/60)=2 小时 → 5.00 × 2 = ¥10.00

## 7. 车牌合法性规则

支持两种格式：
- **普通车牌（7位）**：省份简称 + 市级字母 + 5位字符（如 津B6H920）
- **新能源车牌（8位）**：省份简称 + 市级字母 + 6位字符（如 京AD12345）

省份简称：京津沪渝冀豫云辽黑湘皖鲁新苏浙赣鄂桂甘晋蒙陕吉闽贵粤青藏川宁琼使领
第二位字母：A-H, J-N, P-Z（不含 I, O）

## 8. 数据库结构

应用使用 SQLite 单文件数据库 `data/parking.db`，包含 4 张表：

| 表名 | 用途 |
|------|------|
| USER | 管理员/用户账户（密码 MD5） |
| PARKING | 停车场信息（容量、单价、计数） |
| CAR | 车辆出入库记录（合并汽车表与进出记录表） |
| reservations | 车位预约 |

3 个触发器自动维护数据一致性：
- 新增预约 → P_reserve_count +1
- 删除预约 → P_reserve_count -1
- 容量溢出保护（P_now_count + P_reserve_count ≤ P_all_count）

## 9. 日志

运行日志位于 `logs/parking_ms_YYYYMMDD.log`，按天滚动。包含：
- 登录/注册/改密记录
- 出入库操作记录（含费用）
- 预约创建/清理记录
- 错误与警告

## 10. 故障排查

| 问题 | 解决方案 |
|------|----------|
| 启动白屏 | 检查 `logs/` 日志，常见为 Qt 插件缺失 |
| HyperLPR3 初始化失败 | 检查 config.json 中 models_path 是否包含 .mnn 文件 |
| 摄像头无画面 | 点击「选择本地图片识别」回退 |
| 数据库锁定 | 确保只有一个应用实例运行 |
| 无法登录 | 默认管理员账号在初始化向导中设置；忘记密码可删除 data/parking.db 重新初始化 |

## 11. 演示数据

如需快速演示，可执行 `scripts/seed_demo.sql` 注入测试数据：

```bash
sqlite3 data/parking.db < scripts/seed_demo.sql
```

包含：
- 2 个管理员账户（admin / zhang，密码均为 123456 的 MD5）
- 1 个停车场（新工停车场，100 车位，5元/小时）
- 2 条出入库记录（1 条已结算，1 条在场中）
